用 Hugo + GitHub Pages 的「對應做法」

重點：GitHub Pages 是純靜態，無法做真正的會員登入/鎖文。先把結構與內容體驗做滿，會員可先走「手動流程 + 不公開連結」當過渡，日後若要「真鎖文」再換到支援會員的託管（或自架）。

導覽列三頁

會員公告：content/announcement/_index.md（寫贊助流程說明與支持按鈕連結）。

小說列表：用 taxonomy（例如 categories/series）或建 content/novels/_index.md + 自動列出各作品。

網站問題（FAQ）：content/faq/_index.md。

首頁區塊

Featured：文章 front matter 加 featured: true，首頁模板把 featured 排到最前。

Latest Chapters / Latest：首頁模板抓最新文章（可用 section = posts 或 chapters）。

Tag Cloud：Hugo 內建 taxonomy；PaperMod 也支援標籤頁。

社群與網站地圖

hugo.toml 設定社群連結；sitemap Hugo 會自動產生 sitemap.xml，也可在頁尾放一個「網站地圖」頁連到 /sitemap.xml 或自製索引頁。
Ruby's Garden

贊助／Chatbox

會員公告頁放「前往支持」按鈕（外部贊助連結）。

Chatbox：把提供的嵌入碼貼到 layouts/partials/footer.html 或 PaperMod 的 assets/partials，即可出現左下角對話框（和對方一樣在公告中引導用戶用 Chatbox 留資料）。
Ruby's Garden

「過渡版會員」做法（靜態站可行的範圍）

不公開連結：付款後用 Chatbox 留 email，人工回覆「隱藏路徑」連結；頁面不入導覽、不入索引（front matter 加 private: true 自定義，並把列表模板過濾掉）。

或用外部閱讀清單（如雲端硬碟只分享連結）過渡。

這只是權宜之計，不是真正保護。之後要真正會員鎖文，建議換 Ghost（自架或託管）或 Netlify + Identity（非 GitHub Pages）。

版權宣告

在 content/announcement 或頁尾 partial 寫上你的版權/站規。
